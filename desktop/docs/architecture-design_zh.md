## 1. 产品定位与设计理念

### 1.1 产品定位

Plan Cascade Desktop 是一款面向开发者的桌面端 AI 编程助手，采用**纯 Rust 后端架构**，通过 Tauri 框架实现高效、安全的跨平台桌面应用。

### 1.2 核心设计理念

- **单一真相源 (SSOT)**: 关键状态（如工作流内核阶段）采用单一权威数据源，避免状态不一致
- **事件驱动**: 实时更新通过 Tauri 事件推送，减少轮询开销
- **内存与技能系统**: 支持跨会话持久化记忆和可复用技能模板
- **代码库索引**: 结合 Tree-sitter、FTS5 全文搜索和 HNSW 向量索引的混合搜索能力

## 3. 双模式架构

### 3.1 Chat / Plan / Task 三模式

Plan Cascade Desktop 采用**双模式架构**，支持在同一个应用中切换不同的交互模式：

| 模式 | 用途 | 特点 |
|------|------|------|
| **Chat 模式** | 对话式编程 | 自由对话，LLM 直接执行工具调用 |
| **Plan 模式** | 计划驱动 | 结构化步骤规划，分批执行 |
| **Task 模式** | 任务管理 | 大型任务的 gate 检查和质量门控 |

### 3.2 模式切换机制

- **Chat → Plan/Task**: 基于对话内容智能建议切换
- **Plan/Task → Chat**: 执行完成后可返回对话模式继续
- **Context Handoff**: 模式间上下文通过内核快照传递

### 3.3 Simple 页面 V2 生产路径

Simple 页面（`src/components/SimpleMode`）遵循以下生产约束：

- `workflowKernel.modeSnapshots` 是 Chat/Plan/Task 生命周期的唯一权威来源
- 前端通过 `workflow-kernel-updated` 事件消费内核推送更新
- 工作流卡片仅从类型化 payload (`StreamLine.cardPayload`) 渲染
- Plan/Task 后端会话通过 `workflow_link_mode_session` 明确链接到内核会话

详见 [kernel-design.md](./kernel-design.md)

## 4. 当前能力概览

### 4.1 多 LLM 支持

- 支持多种 LLM 提供商（OpenAI、Anthropic、Ollama 等）
- 可配置的模型选择
- 流式响应支持

### 4.2 代码库智能

| 能力 | 技术实现 |
|------|----------|
| 语义搜索 | HNSW 向量索引 |
| 符号搜索 | FTS5 全文搜索 + BM25 排序 |
| 文件搜索 | FTS5 路径搜索 |
| 类型解析 | LSP 增强层（可选） |

详见 [codebase-index-design.md](./codebase-index-design.md)

### 4.3 Agent 模型

支持多种 Agent 类型：

- **LLM Agent**: 纯 LLM 决策代理
- **Graph Workflow**: 图结构工作流
- **Loop Agent**: 循环执行代理
- **Parallel Agent**: 并行执行代理
- **Conditional Agent**: 条件分支代理

### 4.4 质量门 (Quality Gates)

- 可配置的验证规则
- 步骤输出质量检查
- 重试策略支持

## 5. 核心差异化特点

| 特点 | 说明 | 竞品差异 |
|------|------|----------|
| **纯 Rust 后端** | 无 Python 依赖，单一二进制 | 竞品多依赖外部服务 |
| **工作流内核 SSOT** | 阶段状态单一真相源 | 避免状态漂移 |
| **内存系统 V2** | 跨会话持久化记忆 | 竞品多为会话级 |
| **技能系统** | 可复用任务模板 | 竞品少有内置技能 |
| **混合搜索** | FTS5 + HNSW + LSP | 多维度代码理解 |
| **质量门控** | 步骤级验证 | 更精细的执行控制 |

## 6. 设计哲学说明

### 6.1 可靠性优先

- **内核权威**: 关键状态由内核统一管理，UI 只读不写
- **类型安全**: 全链路 TypeScript + Rust 类型定义
- **错误边界**: 优雅降级，不因局部错误导致整体崩溃

### 6.2 性能优化

- **向量化搜索**: HNSW 将搜索从 O(n) 降至 O(log n)
- **事件驱动**: 推送替代轮询，减少延迟
- **后台索引**: 索引构建不阻塞主线程

### 6.3 扩展性设计

- **Hooks 系统**: Agentic 生命周期钩子支持横切关注点
- **技能格式兼容**: 支持 Plan Cascade SKILL.md、adk-rust .skills/、CLAUDE.md 等多种格式
- **MCP 集成**: 支持 Model Context Protocol 扩展
